# サイト設計図 (Website Design Document)

これから開発を進めるサイトの設計図です。以下の項目について二人で詳細を詰めていきましょう。

## 1. サイトの目的とターゲット (Purpose and Target Audience)
- **目的**: タクシーおよびライドシェア利用者向けのサービス案内と、運転手になりたい方向けの採用・求人案内の両立
- **ターゲットユーザー**: 
  1. タクシーやライドシェアを利用したい一般のお客様
  2. タクシー運転手、または「二種免許なし」で気軽に始められるライドシェアドライバーとして働きたい求職者

## 2. サイトの構成・ページ一覧 (Site Structure / Pages)
- **1. トップページ (Home)**
  - サイトの入り口。「タクシーを利用する」導線と「働く（採用）」導線を明確に分ける。
- **2. お客様向け (For Customers)**
  - サービス・料金案内（タクシー基本料金：初乗り780円、ライドシェア、貸切タクシー、空港送迎など）
  - 料金シミュレーション（出発地と目的地でおおまかな料金がわかるページ）
  - 配車・予約方法（電話番号や、もし配車アプリがあればそのリンク）
  - 車両紹介（普通車、迎賓用、車椅子対応など）
- **3. 採用・求職者向け (For Job Seekers)**
  - グリーンタクシーで働く魅力・先輩インタビュー
  - 働き方の選択（プロのタクシードライバー / 二種免許不要・気楽なライドシェア）
  - 1日の仕事の流れ
  - 募集要項・Q&A
  - エントリーフォーム（応募画面）
- **4. 共通・その他**
  - 会社概要・アクセス
  - お知らせ（News）
  - お問い合わせフォーム
## 3. 主要機能とコンテンツ (Key Features & Content)
- **【お客様向け】料金シミュレーション機能**:
  （将来的に）出発地と目的地を入力すると、距離や時間から概算料金を簡単にシミュレーションできる機能。
- **【求職者向け】魅力が伝わるコンテンツ**:
  「未経験からでも安心」「働きやすい」ということが伝わる写真や、1日のスケジュール紹介。特にライドシェアは「二種免許不要で今すぐ気楽に始められる」というメリットを大きくアピールする構成。
- **【共通】お知らせ（News）機能**:
  「年末年始の営業について」や「採用説明会のお知らせ」などを、今後お客さんが自分で簡単に更新できるような作り（CMSやファイルベースのデータ管理）。
- **【共通】安全なフォーム機能**:
  「お問い合わせ」「採用エントリー」を、スパム対策等を行いながら安全に送信できるフォーム。
## 4. デザインの方向性 (Design Guidelines)
- **テーマカラー**: グリーン（緑色）とホワイト（白色）をベース
- **全体的な印象**: シンプルで清潔感があり、使いやすいデザイン

## 5. 技術スタック (Technology Stack)
- **基本構成**: Vite (React) または Next.js（モダンで高速な表示を実現）
- **スタイリング**: Vanilla CSS または Tailwind CSS（保守しやすく美しいUIを採用）
- **ホスティング**: Vercel または Netlify（無料でSSL対応、セキュリティや環境変数の管理も安全・簡単）
- **理由**: 初心者でも安全にお知らせなどの更新ができ、快適に動作するモダンな構成とするため。
## 6. セキュリティと安全性 (Security & Safety)
- **APIキー等の管理**: もしGoogle Maps APIや外部のメール送信APIなどを使用する場合、**絶対にソースコード（GitHub等の公開リポジトリ）に直接APIキーを書き込まない**（ハードコードしない）こと。
- **環境変数の利用**: 秘密情報は必ず環境変数（`.env`ファイル等、Gitにコミットしないファイルやホスティングサービスの専用設定枠）を利用して管理し、初心者が誤ってネット上にパスワードやキーを公開しても安全な設計とする。
- **フォームの安全性**: エントリーフォームやお問い合わせフォームを作成する場合は、スパム対策（reCAPTCHA等）や悪意のある入力への対策（サニタイズ）を考慮する。
